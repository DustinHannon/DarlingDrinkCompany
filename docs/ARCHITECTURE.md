# Architecture Document — Darling Drink Company Website

## Overview

A statically-exported Next.js single-page application hosted on GitHub Pages. The site has no backend, no database, and no server-side rendering. All content is baked in at build time.

## Key Architectural Decisions

### 1. Static Export (No Server)
- `output: "export"` in `next.config.ts`
- All pages pre-rendered at build time
- No API routes, no SSR, no ISR
- **Why**: Free hosting on GitHub Pages, fast load times, zero server maintenance

### 2. Single Page with Section Navigation
- One `page.tsx` renders all sections in order
- Navbar uses `scrollIntoView({ behavior: "smooth" })` for navigation
- **Why**: All information fits naturally in one scroll flow; no need for route-based navigation for a small service business

### 3. Client Components
- All interactive components use `"use client"` directive
- Server components only at layout level (metadata, fonts)
- **Why**: Framer Motion and react-datepicker require client-side JavaScript; the page is fully static so SSR benefits are minimal

### 4. Conditional basePath
- Dev: `basePath = ""`  (localhost:3000)
- Prod: `basePath = "/DarlingDrinkCompany"` (GitHub Pages subpath)
- Applied via `process.env.NODE_ENV` check in `next.config.ts`
- Components also read basePath for image `src` attributes
- **Why**: GitHub Pages serves under a subpath; local dev needs root path

### 5. Tailwind CSS v4 with @theme
- Custom colors defined in `@theme inline` block in `globals.css`
- Custom utility classes (`.glass`, `.gradient-text`) in plain CSS
- **Why**: Tailwind v4 uses CSS-native theming; custom glassmorphism classes are easier to maintain as plain CSS than as Tailwind plugins

### 6. Form Handling via Formspree
- Form POSTs to Formspree API endpoint
- No backend code needed
- Graceful error fallback shows phone/email
- **Why**: Static site can't process forms; Formspree free tier (50/month) is sufficient for a booking inquiry form

## Data Flow

### Contact Form Submission
```
User fills form → clicks Submit
  → JavaScript FormData constructed (includes formatted date)
  → POST to https://formspree.io/f/{endpoint_id}
  → Formspree sends email to taylor@hellodarlingdesigns.net
  → UI shows success message
  → On error: shows fallback contact info
```

### Build & Deploy
```
Developer pushes to master
  → GitHub Actions triggered
  → npm ci (install deps)
  → npm run build (Next.js static export)
  → /out/ directory uploaded as Pages artifact
  → GitHub Pages deploys new version
  → Live at dustinhannon.github.io/DarlingDrinkCompany/
```

## Performance Considerations

- **Images**: Unoptimized (`images.unoptimized: true`) since Next.js Image Optimization requires a server. Images are served as-is from `/public/images/`.
- **Fonts**: Loaded via `next/font/google` with `display: "swap"` — no FOIT, no external requests after initial load
- **JS Bundle**: Framer Motion is the largest dependency; tree-shaking keeps it reasonable
- **CSS**: Tailwind purges unused styles at build time

## Future Considerations

- **Custom domain**: Update `basePath` to `""` and add CNAME file to `/public/`
- **Image optimization**: Consider a CDN like Cloudflare or switching to Vercel for built-in image optimization
- **CMS integration**: If Taylor wants to manage content, could add a headless CMS (Contentful, Sanity) and fetch at build time
- **Analytics**: Could add Google Analytics or Plausible via the layout
